## 2. Session-aware UI flow

- [ ] 2.1 Implement `QThread`-based worker classes for Detect Spells, Extract Selected, and Extract All Pending jobs in `app/ui/workers.py`; workers must not mutate `SessionState` directly — emit typed signals (`record_extracted(str)`, `progress_updated(int, int)`, `extraction_cancelled()`, `extraction_failed(str, str)`) that main-thread slots use to apply mutations via the existing extraction API; implement cancel via a shared `threading.Event` that the worker checks between records; on Detect Spells completion update the status bar with the count of pending records found; on Extract All Pending completion (normal or after cancel) update the status bar with the count of records extracted; connect all signals to slots that refresh the spell list and document panels
- [ ] 2.2 Implement the document-identity dialog (shown when opening a document whose SHA-256 has no stored metadata; aborting the dialog cancels the open operation and leaves the current session unchanged); implement the different-SHA file-switch prompt — show it only when the active session has at least one confirmed or needs-review record, offering Export (opens export dialog then replaces session), Discard (replaces session immediately), or Cancel (aborts the open); if only pending records exist, replace the session silently without prompting; implement the export-dialog launch (delegates to the export module from `add-export-capabilities`)
- [ ] 2.3 Implement same-SHA reopen (keep in-memory session, refresh only the display path shown in the title bar and status bar) and window-title update on all document-open and path-refresh events (`<filename> — SpellScribe`)
